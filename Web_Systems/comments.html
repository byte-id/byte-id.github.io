Overall

  • I initially faced a slow start with design decisions for layout and look, partly due to the openness of the assignment. I altered it several times to accommodate the content while maintaining a grid-based structure and ensuring responsiveness.
  • While I have some experience with designing more complex websites using databases and JavaScript, I chose to forgo it in favour of refining my CSS skills and work around accessibility.
  • I encountered some challenges, such as maintaining image overlap while keeping responsiveness and getting the desired display of the menu. Some issues could've been easier solved with events in javascript. Additionally, determining optimal picture ratios, sizes, and layout posed its own set of challenges. I could have approached the design with a focus on a fixed screen size, or max or min. Instead, I ended up somewhere in between. While not optimal, it suited the project's purpose as it allowed me to avoid altering some default values for elements multiple times.
  • Throughout the design process, I adapted a logical layout with attention to responsive design principles, and am quite pleased with how it turned out.

Technical

Structure

  • Designed according to the requirements outlined in the assignment on Canvas.
  • Four html documents, one cascading style sheet and numerous images allocated in the websystems directory. The images are located in a separate subdirectory for a clearer structure.
  • Takes you to index page when accessed through the link provided.
  • The website is structured like the illustration below with 4 main sections with different purposes.

Header

  • The header contains a logo, the title of the page and a nested navigation menu with hyperlinks.
  • The header is a flexbox with 3 children. The menu label is positioned to the far right by setting nav margin-left: auto. The logo image has a hyperlink that takes you to the home page.

Navigation

  • I chose a dropdown menu that by default displays as a block. To maximise space and create a horizontal option when the screen size warrants it, I made the nav a flex container with features row-reverse and position: relative. Additionally, I set ul to flex and li to display: inline, within a @media query. For mobile and small screens, the nav width is set to 100%, ensuring it occupies the entire width of the screen.
  • The menu is hoverable, but can also be checked/unchecked. This is achieved through input type="checkbox". Manipulation of the style makes the label clickable without the checkbox being visible. It's set to opacity: 0 rather than display: none to allow it to be selected by pressing tab.

Main

  • The layout the pages are identical, except for the comment page, which was altered to communicate the information more effectivly in a flexbox column, with images center aligned.
  • The container of the main content is a div with the class .content. It houses six child items: three articles and three divs. The layout is structured using a grid with two columns and three rows with slight verical gaps between, visually represented by violet and purple dashed lines. Each article, labelled with classes first, second and third has flex-direction: column, creating a content flip effect. Additionally, they are centrally aligned along the y-axis for a balanced appearance. Images within these sections are centered, both vertically and horizontally and utilises z-index to decide stacking order.
  • The article consists of a heading and a paragraph while the div with the class .pcon(1/2/3) has two illustrative items. These are nested to get the desired layout using grid and flexbox. By using grid, it's easier to add elements seamlessly while catering to a responsive design.
  • For smaller screens, the .content is a flexbox with flex-direction: column. Responsive changes are set with meta viewport through @media queries and relative size values.

Illustrations

  • The images/illustrations in these sections are positioned at the center and overlap slightly when rotated. Rotation is implemented with transform: rotate(xdeg), and nth:child(even/odd) selector is utilised for a symmetrical presentation. On hover, they smoothly zoom in using a transform: scale property and transition. Their responsiveness is achieved through the use of relative values for max-width and min-width: 0 for the items and the container.
  • Tables here serve the same purpose as images and are mostly decorative to underline content. The background is achieved through using a nth-child(even/odd) with background-image: linear-gradient. The rounded border and having selective, minimalistic borders is enabled by border-spacing: 0.
  • The pictures have been resized to fit an aspect ratio of 4:5 for portrait and 4:3 for landscape with a height of 400px. This makes it look more seamless when putting them beside each other.

Footer

  • The footer is a grid. It contains a paragraph with a nested span and a link to the top of the page which is only visible on smaller screens. The grid allows positioning of the link through place-self: end and display: inline, and is styled to look like a button. The span .break allows the paragraph to break.

Selectors and styling

I've chosen to utilize tags with child selectors and groupings more than classes and id's since I find it more effective. Read more on W3schools. I've added some comments in the html (content in css is somewhat chronological) for further inspection. @media queries are positioned at the bottom.

Aesthetics

My design philosophy revolves around minimalism, creating an inviting and classy appearance. It combines clean lines, rounded edges, light colours, and spacious sections with a structured layout. Simplicity, with limited elements and effects ensures clear and concise content presentation, prioritising readability and user-friendliness.

Colour & font

  • The harmonious blend of pastel colours adds a touch of elegance, and also aligns well with the vibrant and inviting feel I wanted to convey.
  • A consistent colour palette that contrasts the text, contributes to a holistic and aesthetically pleasing design.
  • Borders and shadows strategically emphasise content without overwhelming the design.
  • Background colours are strategically applied to text elements to enhance readability and contribute to the design's cohesion.
  • Depending on the context, different fonts may accentuate specific elements (comment page).

Illustrations & interactivity

  • Tables are used as illustrations on multiple pages, but serves the same purposes as images.
  • Images are designed with a white, rounded border for cleaner aesthetics and have interactive zoom effect on hover with a transition for smoother action. They are also tilted and overlap slightly to create a more modern design. Images are sized down to prevent covering the page when CSS is turned off, although resolution may be affected. It also reduces file size and time loading the page.
  • Menu highlights links and label on hover and includes active class for better site navigation, with a dropdown functionality that stays open when checked (clicked).
  • Hover effects are disabled for smaller screens to accommodate touch interactions.

Structure

  • Whitespace and margins provide a clean and spacious layout while maintaining content centrality.
  • The design is responsive, adapting seamlessly to various screens and devices, ensuring a consistent and user-friendly experience.
  • Clear hierarchy aids user navigation, featuring a prominent page title, appropriate headings, a minimalistic header, footer, and menu. The overall design maintains a structured layout with clear divisions and consistent spacing, which enhances visual appeal and usability.
  • Grid layout with an equal number elements allows easy incorporation of new content. Content is center aligned vertically and horizontally

Accessibility

The design strives to be as accessible as possible, catering to visually impaired or blind users who rely on screen readers, as well as individuals with colour blindness. Several features have been incorporated to achieve this:

Screen readers

  • The use of semantic HTML tags enhances the structure and meaning of content.
  • The attribute aria-label and descriptive alt text have been applied to images for improved screen reader comprehension.
  • Site navigation is enabled for keyboard users through tabindex and a checkable menu
  • Content is arranged in the HTML to maintain a chronological order, ensuring that illustrations follow corresponding text in a logical sequence.

Structure

  • Headings are prominently displayed and logically structured to aid navigation.
  • The menu highlights the current and hover states in addition to text for easier identification and navigation.

Font & color

  • Line and font spacing (typography) is increased to enhance readability.
  • Relative text sizes are employed, ensuring compatibility with zoom functionality, and a sans-serif font type enhances readability.
  • Text/background colours have been chosen to provide sufficient contrast with each other to prevent eye strain.
  • Clear lines and well-defined edges, along with appropriate spacing between content elements, contribute to a user-friendly experience.
  • Guidelines to accessibility can be found at w3 and webaim.
